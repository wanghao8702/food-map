# 美食地图 PRD（产品需求文档）

> 版本 v1.0 ｜ 日期 2026-06-03 ｜ 状态：已通过头脑风暴评审，待实现
> 一句话：一张手绘 SVG 世界地图，把《风味人间》纪录片里的美食钉在它的产地上；点一道菜，看插画、介绍、纪录片原文，并能溯源到「是哪一季哪一集」。

---

## 1. 背景与定位

- 这是已有「诗词地图」项目生态的**美食版**。诗词地图把唐诗钉在产地上，美食地图把纪录片里的美食钉在产地上。
- 素材库：`food-map/风味人间/` 下 28 篇《风味人间》文案（第 1–4 季）。文案以 `/地区-地点/` 标记位置，配菜名与叙述段落，结构清晰，是天然的数据源。
- 与诗词地图的关键差异：《风味人间》**跨越全球**（中国 + 法国、西班牙、荷兰等），且**中国部分的美食密度远高于其他地区**。这两点直接决定了地图与交互设计。

## 2. 目标与非目标

**目标**
- 用一张**手绘 SVG 世界地图**承载全球美食，视觉上与手绘美食插画浑然一体。
- 让「一道美食」成为地图上的一个坐标（一菜一钉）。
- 点击后能看到：SVG 插画、美食介绍、纪录片文案、来源（第几季第几集），并可溯源到该集全文。
- 纯前端、单文件、无构建、无后端、无地图服务 key，本地起一个 http server 即可运行。

**非目标（v1 明确不做）**
- 不接入真实瓦片地图 / 高德 / 谷歌地图（海外覆盖弱、且与手绘风格冲突）。
- 不做用户系统、评论、收藏、后端数据库。
- 不做搜索/筛选的复杂功能（v1 先把核心浏览链路跑通）。
- 不做移动端深度适配（v1 以桌面浏览为主，布局保证不破即可）。

## 3. 用户与核心场景

- **目标用户**：喜欢美食纪录片、对「美食 × 地理 × 人文」感兴趣的内容消费者。
- **核心场景**：
  1. 打开网站 → 看到一张手绘世界地图，上面散落着海外美食图钉，中国是一个「中国 · N 道」的聚合气泡。
  2. 点海外某道菜 → 右侧滑出详情，看插画 + 介绍 + 纪录片文案 + 来源。
  3. 点「中国」气泡 → 切换到中国近景图，中国美食按省份铺开。
  4. 在详情里点「查看本集全文」→ 阅读该集完整文案，页面自动定位并高亮当前这道菜的段落。

## 4. 信息架构与数据模型

### 4.1 一菜一条记录（`dishes.json`）

```jsonc
{
  "id": "fw1-01-tuhuangyou",        // 唯一 id：季-集-菜拼音
  "name": "秃黄油拌饭",              // 菜名
  "season": 1,                       // 第几季
  "episode": 1,                      // 第几集
  "episodeTitle": "山海之间",         // 集名
  "region": "江苏·太湖",             // 地点（取自 /地区-地点/ 标记）
  "lng": 120.21,                     // 经度
  "lat": 31.30,                      // 纬度
  "scope": "china",                  // "world" | "china" —— 决定钉在哪一层地图
  "intro": "取大闸蟹蟹黄蟹膏，猪油慢熬，拌入热饭，极致的鲜与肥。", // 美食介绍（一两句）
  "quote": "拆蟹是耐心活……秃黄油拌饭，是江南人对一只蟹最郑重的告别。", // 纪录片文案段落
  "episodeAnchor": "p-tuhuangyou",   // 在该集全文中的段落锚点 id
  "svg": "tuhuangyou.svg"            // 插画文件名（assets/svg/ 下）
}
```

字段说明：
- `scope`：`world` 钉在世界图，`china` 钉在中国近景图。世界图上所有 `china` 记录被聚合成一个气泡。
- `episodeAnchor`：用于「查看本集全文」时定位高亮，对应集全文 HTML/数据里的段落锚点。
- `lng/lat`：v1 由人工为每个地点标注（地点数量有限，逐个查表即可）。同一地点的多道菜可共用坐标，渲染时做微小偏移避免完全重叠。

### 4.2 集全文数据（用于「查看本集全文」）

- 每一集一份结构化全文：保留原文段落顺序，给每个「美食段落」打上与 `episodeAnchor` 对应的锚点 id。
- 集内每道菜在全文中以**锚点标记 + 高亮底色**区分，支持互相跳转。
- v1 数据来源：直接解析 `风味人间/《风味人间1》01 山海之间.md`（用 `/地区-地点/` 标记和加粗菜名切分段落）。

### 4.3 文件结构

```
food-map/
├── index.html              # 单文件应用（地图 + 交互 + 详情面板）
├── dishes.json             # 美食数据（一菜一条）
├── episodes/
│   └── fw1-01.json         # 第1集结构化全文（带段落锚点）
├── assets/
│   ├── svg/                # 每道菜一张扁平填色 SVG 插画
│   ├── world-map.svg       # 手绘世界地图底座
│   └── china-map.svg       # 手绘中国近景地图底座
└── PRD-美食地图.md         # 本文档
```

> 运行方式：在 `food-map/` 下 `python -m http.server`，浏览器访问。因 `dishes.json`/`episodes/*.json` 用 `fetch()` 读取，不能用 `file://` 直接打开。

## 5. 功能需求

### 5.0 整体视觉风格（Claude 暖色调）
- 全站采用 **Claude 暖色调**：暖米纸背景（`#F0EEE6`）、赤陶/珊瑚橙主色（Claude orange `#D97757`，hover 用 `#CC785C`）、暖深棕文字（`#3D3929`）。颜色集中为 CSS 变量统一管理。
- 手绘地图与美食插画沿用同一暖色体系，保证地图、图钉、面板、全文层观感一致。

### 5.1 地图底座（手绘 SVG）
- 复古/古地图质感的手绘世界地图 SVG，暖色调（暖米纸底 + 棕色海岸线），与美食插画、Claude 暖色站点风格统一。
- 单独一张手绘中国近景地图 SVG，能让中国美食按省份大致位置散开。
- 地理为**示意性**，不要求精确投影；坐标 → 屏幕位置通过简单的线性映射（经纬度 → SVG viewBox 坐标）实现。

### 5.2 双层地图交互
- **世界视图（默认）**
  - `scope: "world"` 的美食 → 单个图钉，钉在其经纬度对应位置。
  - 所有 `scope: "china"` 的美食 → 聚合为一个「中国 · N 道」气泡，钉在中国大致中心。
  - 图钉 hover 显示菜名 tooltip；点击打开右侧详情面板。
- **钻取**
  - 点「中国」气泡 → 切换到中国近景图，渲染所有 `china` 美食为图钉。
  - 顶部显示「← 返回世界」，点击回到世界视图。
- **防重叠**：同一坐标的多道菜做小幅放射状偏移；近距离图钉可加细微抖动，保证都可点。

### 5.3 详情面板（右侧滑出）
- 从右侧滑入，地图保持可见。
- 内容块（自上而下）：SVG 插画 → 菜名 → 📍地点 → 🎬第几季·第几集《集名》 → 美食介绍 → 纪录片文案（左边框引用样式）。
- 底部操作：**「查看本集全文」** 按钮。
- 关闭：点面板外区域 / 关闭按钮 / Esc。

### 5.4 本集全文视图
- 点「查看本集全文」→ 展示该集完整文案（可为全屏覆盖层或大面板）。
- 进入时**自动滚动到当前菜的段落锚点**，并以高亮底色标出。
- 集内**每道菜的段落都有锚点标记**（如左侧小图钉/菜名徽标），可点击在不同菜之间跳转，解决「一集多菜文案如何区分」的问题。
- 提供返回详情/关闭。

### 5.5 SVG 美食插画系统
- 风格：**扁平填色**（实色填充 + 少量描边），亲和、好认、有食欲。
- 由 Claude 以代码方式绘制。**先定一套视觉规范 + 复用元件**，再批量产出：
  - 统一画布尺寸 / viewBox、统一描边色与粗细、统一调色板（暖色为主）。
  - 复用基础元件库：碗、盘、鱼、肉块、蔬菜、面条、汤等，组合成具体菜品。
- 目标：几十上百道菜风格保持一致。

## 6. v1 试点范围（务必先做透再扩展）

**只做《风味人间 1 · 第 1 集 山海之间》。** 该集横跨：新疆、内蒙古、安徽、河南、甘肃、四川、江苏、中国台湾，以及西班牙、法国、荷兰——单集即体现「全球 + 中国密集」的全部特征，是理想试点。

候选菜品（约 12–15 道，最终以文案梳理为准）：
蹍转、烙馍、大闸蟹/绒螯蟹、秃黄油拌饭、羊肚菌、冷笋、火腿、搅团/土豆搅团、奶桶肉、熏马肉马肠、大黄鱼、旗鱼。

v1 验收 = 跑通整条链路：
**文案提取 → 坐标标注 → SVG 绘制 → 双层地图渲染 → 详情面板 → 本集全文高亮**，并沉淀出 **SVG 视觉规范 + 复用元件库**。

## 8. 技术栈

- 纯前端单文件 HTML + 原生 JS（延续 `poetry-map-claudedesign` 的无构建风格）。
- 地图与插画均为内联/外链 SVG，无地图 SDK、无 key。
- 数据 `fetch()` 自 `dishes.json` / `episodes/*.json`，本地需 http server。
- 不引入框架（v1 规模无需 React/构建工具）。

## 9. 验收标准（v1）

1. 世界视图正确渲染：海外菜为单钉、中国为聚合气泡。
2. 点中国气泡可切换到中国近景图并显示中国菜钉；可返回世界视图。
3. 点任一图钉，右侧面板正确显示插画/菜名/地点/季集/介绍/文案。
4. 「查看本集全文」能打开该集全文、自动定位并高亮当前菜段落；集内多道菜可区分、可跳转。
5. 第 1 集约 12–15 道菜均有一致风格的扁平填色 SVG 插画。
6. 在 `python -m http.server` 下零报错运行。

## 10. 风险与开放问题

- **SVG 一致性**：菜品多、形态各异，靠"先定规范 + 复用元件"控制；v1 即验证该方法是否可规模化。
- **坐标精度**：手绘地图为示意性，经纬度→屏幕的线性映射可能与手绘海岸线略有偏差，需要按底图微调映射参数。
- **文案版权**：素材为《风味人间》文案，定位为个人/学习用途的非商业展示；如需公开发布需评估授权。
